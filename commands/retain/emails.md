Run the Lifecycle Emails exercise from `~/.claude/skills/retention/SKILL.md` (Stage 1: Lifecycle Emails).

1. Ask which sequence they need (onboarding, engagement, win-back, or all)
2. Ask what their product's aha moment is (the action that makes users stick)
3. Ask about their current email setup (tools, existing sequences, sending frequency)
4. For onboarding: generate the Day 0/1/3/7/14 sequence with subject lines, content outlines, and CTAs
5. For engagement: design trigger-based emails (milestone, usage drop, feature announcement, expansion)
6. For win-back: generate the Day 30/90/180 post-churn sequence
7. Run the email anti-patterns check (noreply, daily overload, no personalization, etc.)
8. Include trigger conditions for each email

Every email should drive ONE action. If there's no clear CTA, don't send it.
